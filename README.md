# 北海道・地域課題リサーチ 説明資料

最終更新：2026年6月11日（v7）

---

## 1. 一言でいうと

北海道の地域課題を「全道共通」と「地域固有」の二層で整理し、課題ごとに**データ・構造解説・取り組み事例・施策の選択肢・調査研究・出典**をひとつのカードにまとめた、公開型の課題マッピングです。誰でもブラウザだけで閲覧でき、すべての情報は公開ソースのURLまで遡れます。

## 2. なぜ作ったか

北海道の地域課題に関する情報は、道庁の計画、省庁の統計、大学の研究、報道、自治体の事例集と、あちこちに分散しています。個々の情報は存在するのに、「課題の全体像を俯瞰し、そこから一次情報へ降りていく」ための入口がありませんでした。

本リサーチは、fladdict氏（深津貴之氏）の「日本が先送りせず解くべき課題（japan-todo）」が提示した「課題カード × 深刻度スコア × 出典」という構造を参照し、それを北海道の文脈——広域分散・積雪寒冷・一次産業依存・札幌一極集中——に合わせて再構成したものです。

## 3. 全体構成

### 二層 × 6連携地域

課題は「全道共通」と「地域固有」の二層で分類しています。地域区分は北海道の政策展開方針と同じ6連携地域を採用しました。

| 区分 | 範囲 |
|---|---|
| 道央 | 石狩・空知・後志・胆振・日高 |
| 道南 | 渡島・檜山 |
| 道北 | 上川・留萌・宗谷 |
| オホーツク | オホーツク管内 |
| 十勝 | 十勝管内 |
| 釧路・根室 | 釧路・根室管内 |

### 8つの社会システム分類

人口・世代・暮らし／一次産業・食・森林／地域・インフラ・交通／医療・福祉・ケア／産業・仕事・移住／環境・エネルギー・防災／観光・交流・共生／デジタル・行政

## 4. 課題カードの読み方

各カードは緊急度・深刻度（各5段階）のスコアを持ち、開くと次の順に情報が並びます。読者の理解の深度に沿った設計です。

1. **要約** — 何が起きているかを2〜3行で
2. **重要性を示すデータ** — 検証済みの数値のみをスタットボックスで表示（例：人口「500万人割れ」、医療「無医地区89・全国最多」、半導体「地価上昇率48.8%・全国1位」）
3. **事実** — 出典に基づく現状の記述
4. **解釈・見立て** — 編集部による構造の読み解き。事実と明確に分離して表示
5. **深掘り解説**（折りたたみ） — 「なぜ起きるか（構造）」「論点」「取り組みを読む視点」の三段で課題の構造を解説
6. **取り組み事例** — 自治体・法人・個人の実例。各事例に出典リンクと記事検索リンク
7. **取りうる施策の例** — 一般的な政策メニューの整理。特定地域への推奨ではないことを明記
8. **調査研究・レポート** — 北大HOPS年報、北海道開発協会、シンクタンク等の研究資料へのリンク
9. **関連情報を検索** — ニュース・道庁資料・国の資料（PDF）・市町村サイトをワンクリックで外部検索
10. **出典・データ** — このカードの根拠となった公開情報の一覧

参照先は三層構造になっています。**出典**（数字の根拠）→**調査研究**（構造を学ぶ）→**外部検索**（今日の最新を追う）。掲載時点の情報と、常に更新される外部情報を役割分担させることで、鮮度の問題を構造的に解いています。

## 5. 機能

- **フィルタ**：社会システム（8分類）× 地域（6＋共通）で絞り込み
- **並び替え**：緊急度／深刻度／取組数
- **深掘りボタン**：カードから構造解説へ直行
- **カード共有**：各カードに固有URL（#課題ID）。リンクを開くと該当カードが自動表示
- **外部検索リンク**：課題ごとに最適化したキーワードで、ニュース・行政資料を直接検索

## 6. 記載ポリシー

本リサーチの信頼性の根拠は、次の編集ルールにあります。

1. **公開情報のみ**。出典は自治体・道庁・省庁・報道・当事者団体の公式発信に限る
2. **一次ソースのURLを確認できない情報は掲載しない**。複数のAIリサーチ結果を取り込む際も、URL裏付けのない数値・事例は採用を見送っている
3. **事実と解釈の分離**。データに基づく記述と編集部の見立てをセクションとして分け、深掘り・施策例には「編集部整理であり推奨ではない」旨を明記
4. スコア（緊急度・深刻度）は相対的な目安であり、絶対評価ではない

## 7. 技術構成と更新

- 依存ライブラリなしの静的HTML＋データファイル（data.js）の2ファイル構成。ビルド不要
- GitHub Pagesでホスティング。**データの追加・修正はdata.jsの編集だけで完結**し、commit履歴がそのまま更新履歴になる
- 課題・事例・出典・統計・施策例・研究参照は、すべてdata.js内の構造化データとして管理

## 8. 想定する使い方

- **地域で活動する人**：自分の地域・分野の課題の全体像と、他地域の先行事例を把握する入口として
- **自治体・支援機関**：施策検討の際の事例リファレンスと、住民・議会への説明素材として
- **研究・学習**：課題の構造解説から調査研究レポートへ降りていく学習導線として
- **メディア・発信者**：記事の背景資料と出典確認の起点として

## 9. 免責と今後

取り組み事例は各主体の公開情報に基づく要約であり、網羅的ではありません。誤りの指摘や追加すべき事例を歓迎し、継続的に更新していく前提の「育てるリサーチ」です。

今後の拡張候補：年次統計の定期更新フロー、事例投稿の受け口（GitHub Issues）、地域別・カテゴリ別の充実。
